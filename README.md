# Switch 2 Master Tracker

A Progressive Web App (PWA) that tracks Nintendo Switch 2 games, consoles, and accessories — with automatic daily game list updates from Wikipedia via GitHub Actions.

## Features

- **162+ games** with format (Physical Cart / Game-Key Card / Digital), regional variants, and release status
- **3 tabs**: Games · Consoles & Bundles (7 models including Japan variants) · Accessories (17 items)
- **Collection tracking**: Own (physical), Own (digital), Wishlist — saved to your browser
- **Filters**: type, status, format, region, ownership, genre
- **PWA**: installs on phone/desktop, works offline via service worker
- **Auto-updates**: GitHub Actions scrapes Wikipedia daily and commits new `data/games.json` — app notifies you when fresh data is available

## Setup (5 minutes)

### 1. Fork or create a GitHub repo

Upload all these files to a new GitHub repo (public or private).

### 2. Enable GitHub Pages

Go to **Settings → Pages → Source** and select:
- **Branch**: `main` (or `master`)
- **Folder**: `/ (root)`

Your app will be live at `https://yourusername.github.io/your-repo-name/`

### 3. Enable GitHub Actions

Actions should be enabled by default. To run the scraper manually:
- Go to **Actions → Update Switch 2 Game Data → Run workflow**

The scraper runs automatically every day at 06:00 UTC.

### 4. (Optional) Generate a proper app icon

The `icons/` folder needs `icon-192.png` and `icon-512.png`.
Quick option: use [favicon.io](https://favicon.io) to generate icons, or use any 192×192 and 512×512 PNG images.

## How auto-update works

```
Wikipedia (List of Nintendo Switch 2 games)
    │
    │  daily at 06:00 UTC
    ▼
GitHub Actions → scripts/scrape.py
    │   • Fetches Wikipedia table
    │   • Infers game type (exclusive / NS2 Edition / third-party)
    │   • Infers genre from title/publisher keywords
    │   • Merges with data/overrides.json (format, region, notes)
    │
    ▼
data/games.json  (committed if changed)
    │
    ▼
Your PWA (fetches on load, service worker caches it)
    │
    ▼
Green banner: "New game data available" → user clicks Refresh
```

## Customising

### Adding format/region data for a new game

Edit `data/overrides.json`:

```json
"Game Title Here": {
  "fmt": "k",
  "region": "var",
  "note": "GKC in Japan; full cart in West"
}
```

**Format codes**: `c` = Physical Cart · `k` = Game-Key Card · `d` = Digital Only · `?` = TBA  
**Region codes**: `ww` = Worldwide · `jp` = Japan Only · `var` = Varies by Region

Keys are matched by normalised substring, so partial titles work.

### Changing the scrape schedule

Edit `.github/workflows/update-games.yml`:

```yaml
- cron: '0 6 * * *'   # daily at 06:00 UTC
# Change to weekly:
- cron: '0 6 * * 1'   # every Monday at 06:00 UTC
```

### Running the scraper locally

```bash
pip install requests beautifulsoup4
python scripts/scrape.py
```

This writes/overwrites `data/games.json`.

## Generating the initial games.json

Before deploying, run the scraper once locally to generate `data/games.json`, then commit it:

```bash
pip install requests beautifulsoup4
python scripts/scrape.py
git add data/games.json
git commit -m "chore: initial game data"
git push
```

## Offline support

The service worker caches the app shell on install. When offline:
- The app opens normally from cache
- Your collection data is in localStorage (always available)
- Game data falls back to the last cached `games.json`

When back online, `games.json` is re-fetched silently in the background.
